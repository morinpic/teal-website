### 14. ⚠️ 「トップに戻る」ボタンの追加
- **一部対応**: ティールカラーの「↑」ボタンが右下に表示され、スクロール後に出現する動作はOK
- **追加対応**: モバイル表示時に料金表（MENUセクション）の価格テキスト（¥8,250、¥6,600、¥5,500等）と重なって読めない。ボタン位置をフローティングRESERVEバーの上に調整するか、右マージンを確保する

### 17. 🔲 ダークモード対応
- ブランドカラー（ティール）と相性の良いダークモード

- 優先度は低いが、モダンなサイトとしての差別化に

### 18. ⚠️ Instagram投稿の埋め込み表示
- **一部対応**: 「準備中」プレースホルダーで仮実装済み
- **残対応**: オーナーのInstagramアカウントURL判明後、実際の投稿を埋め込む
- ※ Xアイコンの削除はP0（項目6）で対応済み
- **実装方法の候補**:
  1. **`react-ig-embed`（推奨）**: APIキー・アクセストークン不要。公開投稿のURLを渡すだけで埋め込み可能。`npm install react-ig-embed` で導入し、`"use client"` のクライアントコンポーネントとして実装する
  2. **Instagram embed.js 手動読み込み**: ライブラリ不使用。Instagramの埋め込みHTMLを `dangerouslySetInnerHTML` で表示し、`useEffect` で `embed.js` を動的ロード。Next.js の `Script` コンポーネントと組み合わせる
  3. **Meta oEmbed Read API（本格的）**: Facebook開発者アカウント・アプリ登録・アクセストークンが必要。フィード全体を動的取得したい場合向け。個人サロンにはオーバースペック
- **進め方**: まずは方法1（react-ig-embed）で最新投稿を数件ピックアップして埋め込む。オーナーのInstagramアカウントURLが判明次第、プレースホルダーを差し替える

### 25. 🔲 モバイルのSTYLEセクションを2カラム化（NEW）
- **現状**: トップページのSTYLE写真6枚がモバイルで1カラム表示。スクロール量が非常に長い
- **改善**: モバイルでも2カラムのグリッドにして、スクロール距離を約半分に短縮

### 26. 🔲 カスタム404ページの作成（NEW）
- **現状**: Next.jsデフォルトの英語404ページ（「404 | This page could not be found.」）
- **改善**: teal.のブランドに合わせたカスタム404ページを作成。ロゴ表示、「ページが見つかりませんでした」の日本語メッセージ、トップページへの導線ボタンを含める

### 27. 🔲 モバイルフッターのナビリンク折り返し（NEW）
- **現状**: フッターの「NEWS / STYLE / MENU / STAFF / BLOG / ACCESS」が1行で並び、「ACCESS」が右端で切れかけている
- **改善**: モバイル表示時にナビリンクを2行に折り返す、またはグリッドレイアウトに変更
