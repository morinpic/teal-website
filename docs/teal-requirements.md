# teal. 公式サイト 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
teal. 公式ウェブサイト制作

### 1.2 クライアント
teal.（美容院）
オーナー / ヘアデザイナー：橋本 貴広（Takahiro Hashimoto）

### 1.3 サイトの目的
- サロンのブランドイメージを伝える
- メニュー・料金情報の提供
- スタイルギャラリーによる技術力のアピール
- ホットペッパービューティーへの予約導線
- お知らせ・ブログによる情報発信
- アクセス情報の提供

### 1.4 背景
Studio.siteで制作を進めていたが、CMSの管理画面がスマートフォンから操作できないという課題があり、新規に構築し直すことを決定。

---

## 2. 技術構成

| 項目 | 選定技術 | 備考 |
|------|----------|------|
| フレームワーク | Next.js (App Router) | 動的ルーティング、SSG/ISR対応 |
| CMS | microCMS (Hobbyプラン・無料) | API 5つ、コンテンツ10,000件、メンバー3名 |
| ホスティング | Vercel (無料プラン) | GitHub連携による自動デプロイ |
| バージョン管理 | GitHub | ソースコード管理 |
| スタイリング | 未定（TailwindCSS推奨） | デザイン実装時に確定 |
| 開発ツール | Claude Code | AI支援による開発 |

### 2.1 CMS設計（microCMS API構成）

| API名 | コンテンツタイプ | 用途 |
|--------|-----------------|------|
| news | リスト形式 | お知らせ＋ブログ（カテゴリフィールドで区別） |
| style | リスト形式 | スタイルギャラリー |
| staff | リスト形式 | スタッフ紹介 |

- MENU（料金表）はコード側で管理（更新頻度が低いため）
- APIを3つに抑えることで、将来の拡張枠を2つ確保
- CMS差し替えに備え、データ取得処理は `lib/microcms.ts` に集約

## 3. サイトマップ・ページ構成

```
teal.
├── トップページ (/)
│   ├── Hero セクション
│   ├── NEWS（最新6件 + リードモアボタン）
│   ├── STYLE（最新6件 + リードモアボタン）
│   ├── MENU（概要 + 詳細ページへのリンク）
│   ├── STAFF（スタッフ一覧）
│   ├── SNS（Instagram埋め込み）
│   └── ACCESS（Google Maps + 店舗情報）
│
├── NEWS 一覧 (/news)
│   ├── ページネーション
│   └── NEWS 詳細 (/news/[slug])
│
├── STYLE 一覧 (/style)
│   ├── ページネーション
│   └── STYLE 詳細 (/style/[slug])
│
├── BLOG 一覧 (/blog)
│   ├── ページネーション（※ NEWSと統合、カテゴリでフィルタ）
│   └── BLOG 詳細 (/blog/[slug])
│
├── MENU 詳細 (/menu)
│   └── カテゴリ別料金表
│
└── STAFF 詳細 (/staff/[slug])
    └── スタッフ個別ページ
```

## 4. ページ詳細仕様

### 4.1 トップページ

#### Hero セクション
- サロン内装の全面写真（背景）
- 「teal.」ロゴを白抜きで中央に配置
- ファーストビューでブランドの世界観を伝える

#### NEWS セクション
- 最新のお知らせを6件表示
- 各項目：日付、タイトル、概要テキスト
- 「READ MORE」ボタンで一覧ページへ遷移

#### STYLE セクション
- 最新のスタイル写真を6件グリッド表示（3×2）
- 各項目：写真、スタイル名
- 「READ MORE」ボタンで一覧ページへ遷移

#### MENU セクション
- 主要カテゴリの概要表示
- 詳細ページへのリンク

#### STAFF セクション
- スタッフ写真＋名前＋肩書きを一覧表示
- 各スタッフの詳細ページへのリンク

#### SNS セクション
- Instagram投稿の埋め込み表示
- Instagram / X へのリンク

#### ACCESS セクション
- Google Maps 埋め込み
- 店舗情報テーブル

### 4.2 NEWS 一覧ページ (/news)
- お知らせ記事の一覧表示
- ページネーション実装
- 各記事をクリックで詳細ページへ

### 4.3 NEWS 詳細ページ (/news/[slug])
- タイトル、日付、本文
- 前後の記事へのナビゲーション
- 一覧に戻るリンク

### 4.4 STYLE 一覧ページ (/style)
- スタイル写真のグリッド表示
- ページネーション実装
- 各スタイルをクリックで詳細ページへ

### 4.5 STYLE 詳細ページ (/style/[slug])
- スタイル写真（大サイズ）
- スタイル名、担当スタイリスト、施術メニュー等
- 一覧に戻るリンク

### 4.6 BLOG 一覧ページ (/blog)
- microCMS の news API からカテゴリ「ブログ」でフィルタ
- ページネーション実装

### 4.7 BLOG 詳細ページ (/blog/[slug])
- NEWS詳細と同様の構成

### 4.8 MENU 詳細ページ (/menu)
- カテゴリ別の料金表を全て表示
- カテゴリ：CUT / COLOR / PERM / TREATMENT / OTHER

### 4.9 STAFF 詳細ページ (/staff/[slug])
- スタッフ写真、名前、肩書き
- 自己紹介文
- 得意なスタイル・メッセージ等

## 5. デザイン仕様

### 5.1 ブランドカラー
- **プライマリ（ティール）**: セクション背景、見出しアクセントに使用
- **ダークカラー**: #231815（ロゴ色、テキスト）
- **ホワイト**: #FFFFFF（背景、コントラスト）

### 5.2 デザインコンセプト
- ミニマル・モダン
- SHACHUのレイアウト構成を参考に、teal.独自のティールカラーとセクション背景の交互切り替えを活かす
- 余白を広くとり、写真で魅せるデザイン
- 見出しは英語（大文字）＋ 日本語サブタイトル

### 5.3 ロゴ
- SVGフォーマットで提供済み
- ヘッダー：ダークカラー版
- Hero / フッター：ホワイト版

### 5.4 レスポンシブ対応
- モバイルファースト
- ブレイクポイント：SP（〜768px）、タブレット（769px〜1024px）、PC（1025px〜）

## 11. 非機能要件

### 11.1 パフォーマンス
- Lighthouse スコア 90点以上を目標
- 画像は WebP / AVIF 形式で最適化
- Next.js の Image コンポーネントによる遅延読み込み

### 11.2 SEO
- メタタグ（title / description）の適切な設定
- OGP画像の設定
- 構造化データ（LocalBusiness）の実装
- sitemap.xml / robots.txt の生成

### 11.3 アクセシビリティ
- セマンティックなHTML構造
- 画像への alt テキスト
- キーボードナビゲーション対応

### 11.4 セキュリティ
- microCMS API キーの環境変数管理
- HTTPS 通信（Vercel標準）

## 12. 運用・保守

### 12.1 コンテンツ更新フロー
- お知らせ / ブログ / スタイル写真 → microCMS管理画面からスマホで更新
- メニュー・料金 → GitHubリポジトリの更新（Claude Codeで対応）
- スタッフ情報 → microCMS管理画面から更新

### 12.2 デプロイフロー
- GitHub main ブランチへのプッシュ → Vercel 自動デプロイ
- microCMS Webhook → ISR による自動再生成
